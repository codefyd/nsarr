# دليل ملء البيانات الافتراضية

هذا الدليل يشرح كيفية استخدام دوال ملء البيانات الافتراضية للاختبار والعرض.

## 📋 محتويات البيانات الافتراضية

### 1. الطلاب (5 طلاب)
- **أحمد محمد علي** - حلقة الفاروق - 5 أجزاء
- **فاطمة خالد محمود** - حلقة أبي بن كعب - 8 أجزاء
- **سارة عمر حسن** - حلقة ابن كثير - 12 جزء
- **محمد سالم ناصر** - حلقة الفاروق - 3 أجزاء
- **نور يوسف محمد** - حلقة أبي بن كعب - 6 أجزاء

### 2. الطلبات الواردة (3 طلبات)
- طلب تسجيل جديد (قيد الانتظار)
- طلب تعديل بيانات (قيد الانتظار)
- طلب تسجيل مقبول (تم معالجته)

### 3. ملاحظات المعلمين (4 ملاحظات)
- ملاحظات عن الطلاب من المعلمين

### 4. الإنذارات التعليمية (3 إنذارات)
- إنذار ضعف حفظ
- إنذار غياب
- إنذار سلوك

### 5. الإنذارات الإدارية (2 إنذار)
- إنذار تأخر
- إنذار غياب

---

## 🚀 كيفية الاستخدام

### الخطوة 1: إضافة ملف seed-data.gs

1. افتح Google Apps Script
2. انقر على **+** لإضافة ملف جديد
3. اختر **البرنامج النصي**
4. سمّه: `seed-data`
5. انسخ محتوى ملف `seed-data.gs` بالكامل
6. الصقه في المحرر
7. احفظ المشروع

### الخطوة 2: تشغيل دالة ملء البيانات

#### الطريقة الأولى: ملء البيانات مباشرة

1. من قائمة الدوال، اختر: `seedAllData`
2. انقر على زر التشغيل (▶)
3. ستظهر رسالة النجاح في السجل

#### الطريقة الثانية: إعادة تعيين النظام بالكامل

إذا أردت حذف جميع البيانات الحالية وملء البيانات الافتراضية من جديد:

1. من قائمة الدوال، اختر: `resetSystem`
2. انقر على زر التشغيل (▶)
3. ستتم إعادة تعيين النظام بالكامل

## 🔧 الدوال المتاحة

### 1. `seedAllData()`
**الوصف:** ملء جميع الصفحات بالبيانات الافتراضية

**الاستخدام:**
```javascript
seedAllData()
```

**النتيجة:** 
- إضافة 5 طلاب
- إضافة 3 طلبات
- إضافة 4 ملاحظات
- إضافة 3 إنذارات تعليمية
- إضافة 2 إنذار إداري

---

### 2. `seedStudents()`
**الوصف:** ملء صفحة الطلاب فقط

**الاستخدام:**
```javascript
seedStudents()
```

---

### 3. `seedRequests()`
**الوصف:** ملء صفحة الطلبات الواردة فقط

**الاستخدام:**
```javascript
seedRequests()
```

---

### 4. `seedTeacherNotes()`
**الوصف:** ملء صفحة ملاحظات المعلمين فقط

**الاستخدام:**
```javascript
seedTeacherNotes()
```

---

### 5. `seedEducationalWarnings()`
**الوصف:** ملء صفحة الإنذارات التعليمية فقط

**الاستخدام:**
```javascript
seedEducationalWarnings()
```

---

### 6. `seedAdministrativeWarnings()`
**الوصف:** ملء صفحة الإنذارات الإدارية فقط

**الاستخدام:**
```javascript
seedAdministrativeWarnings()
```

---

### 7. `clearAllData()`
**الوصف:** حذف جميع البيانات (ما عدا الرؤوس)

**الاستخدام:**
```javascript
clearAllData()
```

**تحذير:** ⚠️ هذه الدالة تحذف جميع البيانات بشكل نهائي

---

### 8. `resetSystem()`
**الوصف:** إعادة تعيين النظام بالكامل

**الخطوات:**
1. حذف جميع البيانات
2. إعادة إنشاء جميع الجداول
3. ملء البيانات الافتراضية

**الاستخدام:**
```javascript
resetSystem()
```

**تحذير:** ⚠️ هذه الدالة تحذف جميع البيانات الحالية


2. ستظهر جميع الرسائل والأخطاء

## ⚠️ تحذيرات مهمة

### 1. البيانات الافتراضية مثال فقط
البيانات الموجودة في `seed-data.gs` هي مثال فقط. يمكنك تعديلها حسب احتياجاتك.

### 2. حذف البيانات نهائي
عند تشغيل `clearAllData()` أو `resetSystem()`:
- سيتم حذف جميع البيانات الحالية
- لا يمكن استرجاع البيانات المحذوفة
- تأكد من عمل نسخة احتياطية قبل الحذف

### 3. الأداء
عند ملء كمية كبيرة من البيانات:
- قد تستغرق الدالة بعض الوقت
- لا تغلق المتصفح أثناء التنفيذ
- انتظر حتى تظهر رسالة النجاح

## 🛠️ تخصيص البيانات

### تعديل بيانات الطلاب

افتح ملف `seed-data.gs` وعدّل دالة `seedStudents()`:

```javascript
function seedStudents() {
  const students = [
    {
      'اسم الطالب ثلاثي': 'اسم جديد',
      'هوية الطالب': '12345678',
      // ... باقي البيانات
    },
    // أضف طلاب جدد هنا
  ];
  // ...
}
```

### إضافة طلاب جدد

أضف كائن جديد في مصفوفة `students`:

```javascript
{
  'id': معرف('student'),
  'اسم الطالب ثلاثي': 'اسم الطالب',
  'هوية الطالب': '12345678',
  // ... باقي البيانات
}
```

## 🆘 استكشاف الأخطاء

### المشكلة: رسالة خطأ عند التشغيل

**الحل:**
1. تأكد من تشغيل `setupSpreadsheet()` أولاً
2. تأكد من أن جميع الجداول موجودة
3. تحقق من السجلات للتفاصيل

### المشكلة: البيانات لم تظهر

**الحل:**
1. حدّث الصفحة (F5)
2. تحقق من أن الجداول تحتوي على البيانات
3. جرّب تشغيل `seedAllData()` مرة أخرى

### المشكلة: البيانات مكررة

**الحل:**
1. شغّل `clearAllData()` أولاً
2. ثم شغّل `seedAllData()` مرة أخرى
